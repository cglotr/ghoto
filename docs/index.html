Ghoto

🚀

Upload photos to Google Photos — fast.

Ghoto is a CLI tool that uploads photos and videos from a local directory to a Google Photos album, then removes local copies after a successful upload.

View on GitHub Releases

Features

⚡

Concurrent uploads

Splits files across up to 10 parallel workers for fast batch uploads.

🔁

Auto retry

Retries failed uploads up to 10 times with incremental back-off.

🗂️

Album management

Finds or creates the target album automatically before uploading.

🧹

Auto cleanup

Deletes local files only after confirming a successful upload.

🔍

Dry run mode

Preview what would be uploaded without making any changes.

🔐

OAuth 2.0

Authenticates securely with Google via standard OAuth 2.0 flow.

Supported file types

.jpg .mp4

Files starting with . are automatically skipped. .dng and .lrv sidecar files are cleaned up after the main upload.

Getting started

1

Clone & build

git clone https://github.com/cglotr/ghoto.git
cd ghoto
go build -o ghoto .
2

Add Google credentials

Place your Google OAuth 2.0 credentials.json in the project root. The first run will open a browser to complete authentication.

3

Run

# Upload photos from ~/Pictures/Insta360 to the "Insta360" album
./ghoto -dir ~/Pictures/Insta360 -album Insta360

# Dry run — no uploads, no deletions
./ghoto -dir ~/Pictures/Insta360 -dryrun

CLI flags

-album   string   Album name to upload to  (default "Insta360")
-dir     string   Photo directory to upload
-dryrun  bool     Non-mutative dry run mode
